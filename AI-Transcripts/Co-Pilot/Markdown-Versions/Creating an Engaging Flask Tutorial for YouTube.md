### Differences
- **Structure**: Procedural programming is linear and straightforward, while OOP is modular and based on objects.
- **Data Management**: Procedural programming uses global data that can be accessed by any function, whereas OOP encapsulates data within objects, promoting data hiding and abstraction.

### Similarities
- **Control Flow**: Both paradigms use control structures like loops, conditionals, and functions/methods.
- **Reusability**: Both paradigms promote code reusability, but OOP takes it a step further with inheritance and polymorphism.

### Advantages of Procedural Programming
- **Simplicity**: Easier to understand and implement for small projects.
- **Performance**: Generally faster for simple, linear tasks.

### Disadvantages of Procedural Programming
- **Maintenance**: Harder to manage and scale for large projects.
- **Global Data**: Increased risk of data corruption due to global data access.

### Advantages of OOP
- **Modularity**: Easier to manage and maintain due to modular design.
- **Reusability**: Promotes code reuse through inheritance and polymorphism.

### Disadvantages of OOP
- **Complexity**: More complex to understand and implement for beginners.
- **Performance**: Slightly slower due to the overhead of object management.
