---

### 🔹 Step 1: Store the initial meeting date

### 🔹 Step 3: Calculate the number of weeks since the initial meeting
Use an **expression** in a Compose action:

```plaintext
div(
    sub(
        ticks(CurrentDate),
        ticks(variables('InitialMeetingDate'))
    ),
    mul(14,864000000000)
)
```

Explanation:
- `ticks()` gives you the number of 100‑nanosecond intervals since 01‑01‑0001.
- Subtracting the ticks gives you the difference between the two dates.
- `mul(14,864000000000)` = number of ticks in 14 days (bi‑weekly).
- `div()` gives you how many full 2‑week cycles have passed.

---

### 🔹 Step 4: Calculate the next meeting date
Now add one cycle to get the **next meeting**:

```plaintext
addDays(
    variables('InitialMeetingDate'),
    mul(

                    ticks(CurrentDate),
                    ticks(variables('InitialMeetingDate'))
                ),
                mul(14,864000000000)
            ),
            1
        ),
        14
    )
)
```

This expression:
- Finds how many cycles have passed.
- Adds one to move to the next cycle.
- Multiplies by 14 days.

When you use the **“Add a row into a table”** action for Excel, the **Row** parameter must be passed as a **JSON object** where each property name exactly matches the column headers in your Excel table, and the values are the data you want to insert.  

---

### ✅ Format Example
Suppose your Excel table has these columns:

| ID | Name | Date | Status |

Your **Row** parameter should look like this:

```json
{
  "ID": "123",
  "Name": "Hasan",
  "Date": "2025-12-02",
  "Status": "Pending"
}
```

- **Keys** (`ID`, `Name`, `Date`, `Status`) must match the column names in the Excel table exactly (case-sensitive, no extra spaces).
- **Values** are the data you want to insert into that row.

---

### ⚙️ Notes
- If your table has spaces in column names (e.g. `Start Date`), you must use the exact name:  
  ```json
  { "Start Date": "2025-11-17" }
  ```
- Dates should be in ISO format (`yyyy-MM-dd`) to avoid regional formatting issues.
- If you’re pulling values dynamically (e.g. from a form), you can insert them with expressions like:  
  ```json
  {
    "Name": "@{outputs('Get_response_details')?['body/Name']}",
    "Date": "@{utcNow()}"
  }
  ```
